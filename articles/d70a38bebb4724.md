## TL;DR

Gentooを触っている時点で初心者じゃないだろうというツッコミが飛んできそうですが、普段私自身mainlineカーネルを使っており簡単に設定できないかを考えていました。
只今執筆時点で最新版はLinux 7.2.0-rc1が出ていますが、これを初心者がいちから設定しようとすると「どこを有効化して良いのか」や「どこを無効化してはいけないのか」全く分からないと思います。
最悪カーネルが起動せず最初からやり直しになる可能性を避けるため、簡単に設定する方法を紹介したいと思います。

## 方法

まず、（詳しく書くと今回の範疇外でとても長くなるため書きませんが）インストールの方法を記したドキュメント通りgentoo-kernel-binを入れる方法でGentooをインストールしてください。（initシステムはお好きな物をお使いください。SystemdであってもOpenRCであっても今回の方法は変わりません。）
Gentooが完成したら次に実際に次のコマンドでgit-sourcesパッケージ（mainlineカーネル）を入れていきます。

```bash
sudo emerge --ask sys-kernel/git-sources
```

この時、git-sourcesに付属しているUSEフラグは気にしなくても構いません。無くても動きます。
次にカーネルを切り替えます。今gentoo-kernel-binのカーネルとgit-sourcesのカーネルが入っていると思いますが、次で確認できます。（事前にapp-admin/eselectパッケージをインストールしておきましょう。）

```bash
sudo eselect kernel list
```

そうすると

```bash
Available kernel symlink targets:
  [1]   ＜gentoo-kernel-binのカーネル＞ *
  [2]   linux-7.2-rc1
```

のようなものが現れたっと思います。次に

```bash
sudo eselect kernel set 2
```

などでgit-sourcesの方にカーネルを切り替えてください。また、`/usr/src`以下で`ls -la`を実行しlinuxのsymlinkがgit-sourcesで入れたカーネルに向いていることを確認してください。この確認を怠ると他のカーネルを編集することになり最悪他のカーネルを壊します。
ついで`/usr/src`以下の`linux`の中に入り、gentoo-kernel-binのカーネルの設定を引き継ぎます。

```bash
sudo zcat /proc/config.gz | sudo tee .config >/dev/null
```

これでビルドすべきカーネルの設定が出来ました...と言いたいところですが、このまま`sudo make olddefconfig`を実行するとカーネルの署名で失敗します。このため、以下を設定します。

```bash
sudo ./scripts/config --set-str SYSTEM_TRUSTED_KEYS ""
sudo ./scripts/config --set-str SYSTEM_REVOCATION_KEYS ""
sudo ./scripts/config --set-str MODULE_SIG_KEY "certs/signing_key.pem"
sudo ./scripts/config --disable MODULE_SIG_FORCE
sudo ./scripts/config --disable MODULE_SIG_ALL
```

また、以下の設定は入れても入れなくても構いませんが、ビルド後カーネルの名前がそのまま入るのであえて入れています。デフォルトはgentoo(or gentoo-dist)になっていると思いますが空にしておきます。デフォルトでは`<カーネル名>-gentoo-dist`(or `<カーネル名>-gentoo`)となります
お好きなものに変えていただいても構いません。

```bash
sudo ./scripts/config --set-str LOCALVERSION ""
sudo ./scripts/config --disable LOCALVERSION_AUTO
```

設定が終わったら先程の設定の確認をします。

```bash
grep -E 'CONFIG_(LOCALVERSION|LOCALVERSION_AUTO|MODULE_SIG_KEY|MODULE_SIG_FORCE|MODULE_SIG_ALL|SYSTEM_TRUSTED_KEYS|SYSTEM_REVOCATION_KEYS)' .config
```

カーネルの設定が完了したら次はようやくカーネルのコンパイル・ビルドです。少々時間がかかりますのでコーヒーを飲んだりしてゆっくり過ごしましょう。
PC環境にもよりますが数時間かかることもあります。
終わり次第モジュールのインストールも行いましょう。

```bash
sudo make -j(nproc)
sudo make modules_install
```

また、私はsystemd-bootで構成しており`/efi`直下にカーネルが散らばるのが好きではないのでわざと`/efi/gentoo`ディレクトリを作りその下にカーネルを置くようにしています。

```bash
kver="$(make -s kernelrelease)"
sudo mkdir -p "/efi/gentoo/$kver"
sudo cp arch/x86/boot/bzImage "/efi/gentoo/$kver/linux"
sudo cp System.map "/efi/gentoo/$kver/System.map"
sudo cp .config "/efi/gentoo/$kver/config"
```

そして`/efi/loader`直下にはsystemd-bootの設定が入っており、ここのloader.confでは以下のように設定しています。

```text
title Gentoo Linux git
version 7.x.x
linux /gentoo/7.x.x/linux
options root=UUID=<root-partition-uuid> rootfstype=ext4 rw
```

普段はbtrfs+LUKS Encryptionでgentooを構成することが多いですが、分かりやすいようにext4で書いています。
